haveibeenexploited.com

Exploited Ethereum Contract Database

After taking a course in Blockchain by professor Wu-Chang Feng in the Computer Science department at Portland State University, I sought a way to improve security in the ethereum blockchain in a small way.

In short Solidity, the language used to write smart contracts for the Ethereum blockchain, can easily contain insecurities depending on version and developer skill level. Researchers have discovered ways to find exploitations (e.g., symbolic execution engines). This research is rarely seen or used by the average user of smart contracts, but contains valuable information that could inform a user whether a contract is safe to use or not. Our project lowers the barrier to entry to this information. All a user needs is to insert the hexidecimal address given to a contract to tell if it is secure or not.

Try it out! Visit the website haveibeenexploited.com and enter 0xe446754f34a00c726a6ecd5d711fe6ac91dd44aa into the search bar to see results!
